## 一、项目概述

### 1.1 项目名称
**名称规范记录智能体（Name Authority Record Agent）**

### 1.2 项目目标
构建一个自动化智能体，从互联网（知网、期刊官网、学术数据库）搜索并采集学者信息，经结构化提取后，输出与 dedup-librecord 系统兼容的规范记录 Excel。

### 1.3 应用场景
- 图书馆名称规范档建设
- 学术机构人才画像
- 科研成果管理与去重匹配

## 二、系统架构

### 2.1 V2 架构设计

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  ① 采集层     │ →  │  ② Excel存储  │ →  │  ③ LLM提取层  │
│  collector.py │    │  pandas/Excel │    │  extractor.py │
│  (CDP + XHR)  │    │  中间格式     │    │  (DeepSeek)   │
└──────┬───────┘    └──────────────┘    └──────────────┘
       │
  ┌────┴────────┐
  │ CDP Proxy   │ ← Node.js WebSocket → REST 转换
  │ :3456       │
  └────┬────────┘
       │
  ┌────┴────────┐
  │ Chrome 浏览器 │ ← 远程调试模式
  └─────────────┘
```

**V2 核心理念：采集与提取解耦。** 第一步全量采集原始文本到 Excel，第二步再用 LLM 从 Excel 中提取结构化字段。这样 LLM 调用次数可控，各步骤可独立调试。

### 2.2 V1 架构（已归档）
V1 采用紧耦合流水线，在内存中实时提取→合并→输出。优势是端到端自动化，劣势是调试困难、LLM 调用频繁。

### 2.3 技术栈

| 层级 | 技术选型 | 用途 |
|------|----------|------|
| 浏览器控制 | Chrome DevTools Protocol (CDP) | 操控浏览器执行搜索、点击、翻页 |
| 协议转换 | Node.js CDP Proxy (localhost:3456) | WebSocket CDP → REST API |
| 后端语言 | Python 3.13 | 采集器、提取器、Web 服务 |
| HTTP 请求 | urllib.request + XMLHttpRequest (JS) | 知网搜索 API 调用 |
| 数据处理 | pandas, openpyxl | Excel 读写 |
| LLM 提取 | DeepSeek API (OpenAI-compatible) | 非结构化文本→结构化字段 |
| Web 服务 | Flask (:8820) | V1 管理界面 |

## 三、核心功能模块

### 3.1 论文采集器（collector.py）

**功能**: 按作者名搜索知网 → 翻页采集 → 打开论文全文 → 提取元数据 → 写入 Excel

**三大采集来源**（按优先级）:

| 来源 | 触发条件 | 提取内容 |
|------|----------|----------|
| HTML 阅读 | 论文有 HTML 全文链接 | 标题、摘要、关键词、基金、作者简介、收稿日期 |
| 知网详情页 | 无 HTML 但有 kcms2 链接 | 标题、作者机构、摘要、关键词、基金、专辑、专题、分类号 |
| XHR 元数据 | 以上均不可用 | 搜索列表中的元数据文本 |

**关键参数**:
- `--max N`: 限制 HTML 阅读打开篇数（默认 20）
- 总论文上限: 200 篇
- 翻页上限: 10+ 页（每页 20 篇）

### 3.2 翻页机制（v2.2 重大突破）

知网的翻页是此项目最具挑战性的技术难题。经深入逆向分析知网 JS 源码（kns.brief.min.js），发现了完整的翻页参数体系：

**第 1 页（初始搜索）**:
```
POST /kns8s/brief/grid
boolSearch=true
QueryJson=<完整搜索条件JSON>
```

QueryJson 必须包含的字段（缺一不可）:
- `Resource: "CROSSDB"`
- `Classid: "WD0FTY92"`
- `Products`: 全部数据库代码
- `KuaKuCode`: 跨库编码
- `ExScope`: 检索范围
- `Rlang`: 语言
- `SearchType`: 搜索类型
- `QNode`: 搜索条件树（作者=xxx）

**第 2+ 页（翻页）**:
```
POST /kns8s/brief/grid
boolSearch=false
QueryJson=<查询JSON(pageNum=N)>
pageNum=N
pageSize=20
turnpage=<从第1页响应中提取的令牌>
sortField=PT
sortType=desc
dstyle=listmode
...（共18个参数）
```

**关键发现**:
1. `turnpage` 令牌从第 1 页 HTML 响应的 `<input id="hidTurnPage">` 中提取
2. 翻页需要 18 个完整的 POST 参数，缺任何一项都会返回"无数据"或第 1 页内容
3. `boolSearch=true` 时，知网强制将 CurPage 设为 "1"，所以翻页必须用 `boolSearch=false`

### 3.3 LLM 提取器（extractor.py）

**功能**: 将非结构化文本（论文摘要、作者简介、官网页面）通过 LLM 提取为 16 个结构化字段。

**16 个字段定义**:
| 字段 | RDA 优先级 | 提取来源 |
|------|-----------|----------|
| 姓名 | 基础 | 论文署名/搜索条件 |
| 别名 | 低 | 拼音/曾用名 |
| 性别 | 中 | 论文简介 |
| 民族 | 低 | 论文简介/官网 |
| 学历 | 中 | 论文简介/官网 |
| 国籍 | 低 | 默认中国 |
| 生卒年 | 极高 | 论文简介/百科 |
| 籍贯 | 高 | 论文简介/官网 |
| 活动领域 | 高 | 研究方向/关键词 |
| 受教育机构 | 中 | 毕业院校 |
| 在职单位 | 极高 | 论文署名/官网 |
| 职业 | 中 | 职称/职务 |
| 职称 | 中 | 教授/副教授等 |
| 电子邮箱 | 低 | 论文/官网 |
| 著作实体 | 高 | 成果列表 |
| 控制号 | — | 外部系统 ID |

**多来源优先级**（防数据污染）:
```
LLM自适应提取 (20) > 官网个人页 (10) > 期刊官网 (9) > 百度百科 (8) >
论文HTML全文 (7) > 论文摘要页 (5) > 搜索引擎摘要 (3) > 搜索条件 (1)
```

### 3.4 提取策略（record.py）

**防合著者污染**: 仅从目标作者姓名所在的句子中提取个人信息，不搜索全文。
**教育/工作经历分离**: 分别匹配，互不污染。
**增量合并**: 多来源数据按优先级合并，高优先级不覆盖已有字段。

### 3.5 Web 管理界面

V1 提供 Flask Web 服务（:8820），包含:
- 搜索启动面板
- 流水线状态监控
- 记录查看与编辑
- Excel 导出

## 四、技术难点与解决方案

| 难点 | 尝试方案 | 最终方案 |
|------|----------|----------|
| **知网翻页** | 修改 pageNum → briefRequest 回传 → setQueryJson 状态同步 → ... | **完整 QueryJson + turnpage 令牌 + 18 参数** |
| **页面激活** | 手动搜索激活 → 模拟点击分类标签 | **手动设置全部搜索参数（无需 UI 激活）** |
| **反爬检测** | DrissionPage → Playwright → Stealth.js | CDP 代理 + `navigator.webdriver=false`（已确认无检测） |
| **DOM 注入** | innerHTML 注入 briefBox → 丢失隐藏 input | **Python 端从原始 HTML 正则提取（不依赖 DOM）** |
| **HTML 编码** | 全角冒号导致 SyntaxError | 替换为 ASCII 等效字符 |
| **Excel 文件锁** | 同文件名冲突 | 文件名加时间戳 |

```bash
node C:/Users/Administrator/.claude/skills/web-access/scripts/cdp-proxy.mjs
```


1. CDP 代理运行在 localhost:3456
2. Chrome 浏览器以远程调试模式运行
