> 版本: v2.3 | 日期: 2026-05-17 | 作者: 赵一冉

## 一、项目概述

### 1.1 项目名称
**名称规范记录智能体（Name Authority Record Agent）**

### 1.2 项目目标
构建一个自动化智能体，从互联网（知网、期刊官网、学术数据库）搜索并采集学者信息，经结构化提取后，输出与 dedup-librecord 系统兼容的规范记录 Excel。

### 1.3 应用场景
- 图书馆名称规范档建设
- 学术机构人才画像
- 科研成果管理与去重匹配

## 二、系统架构

### 2.1 V2 架构设计

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  ① 采集层     │ →  │  ② Excel存储  │ →  │  ③ LLM提取层  │
│  collector.py │    │  pandas/Excel │    │  extractor.py │
│  (CDP + XHR)  │    │  中间格式     │    │  (DeepSeek)   │
└──────┬───────┘    └──────────────┘    └──────────────┘
       │
  ┌────┴────────┐
  │ CDP Proxy   │ ← Node.js WebSocket → REST 转换
  │ :3456       │
  └────┬────────┘
       │
  ┌────┴────────┐
  │ Chrome 浏览器 │ ← 远程调试模式
  └─────────────┘
```

**V2 核心理念：采集与提取解耦。** 第一步全量采集原始文本到 Excel，第二步再用 LLM 从 Excel 中提取结构化字段。

### 2.2 技术栈

| 层级 | 技术选型 | 用途 |
|------|----------|------|
| 浏览器控制 | Chrome DevTools Protocol (CDP) | 操控浏览器执行搜索、点击、翻页 |
| 协议转换 | Node.js CDP Proxy (localhost:3456) | WebSocket CDP → REST API |
| 后端语言 | Python 3.13 | 采集器、提取器、Web 服务 |
| HTTP 请求 | urllib.request + XMLHttpRequest (JS) | 知网搜索 API 调用 |
| 数据处理 | pandas, openpyxl | Excel 读写 |
| LLM 提取 | DeepSeek API (OpenAI-compatible) | 非结构化文本→结构化字段 |
| Web 服务 | Flask (:8820) | V1 管理界面 |

## 三、核心功能模块

### 3.1 论文采集器（collector.py v2.3）

**流程**: 搜索作者 → XHR 翻页 → 三种来源提取 → 写入 Excel

| 来源 | 触发条件 | 提取内容 |
|------|----------|----------|
| HTML 阅读 | 论文有 HTML 全文链接（默认最多打开 20 篇） | 标题、摘要、关键词、基金、作者简介、收稿日期 |
| 知网详情页 | 无 HTML 但预提取了 kcms2 链接 | 标题、作者机构、摘要、关键词、基金、专辑、专题、分类号 |
| XHR 元数据 | 以上均不可用 | 搜索列表中的元数据文本（兜底） |

**v2.3 实测数据（张伟，2026-05-17）**:
- 搜索结果: 76,378 条
- 翻页: 11 页, 217 篇标题
- 详情链接: 每页 19-20 个, 跨页汇总至采集阶段
- 最终输出: 200 篇 Excel
- 来源分布: HTML 阅读 ~12 篇, 知网详情页 ~180 篇, XHR 元数据少数

### 3.2 翻页机制

知网的翻页是此项目最具挑战性的技术难题。经深入逆向分析知网 JS 源码（kns.brief.min.js），发现了完整的翻页参数体系。

**第 1 页（初始搜索）**:
```
POST /kns8s/brief/grid
boolSearch=true
QueryJson=<完整搜索条件JSON>
```

QueryJson 必须包含的字段:
```
Resource: "CROSSDB"
Classid: "WD0FTY92"
Products, KuaKuCode, ExScope, Rlang, SearchType, QNode
```

**第 2+ 页（翻页）** — 共 18 个参数:
```
POST /kns8s/brief/grid
boolSearch=false
QueryJson=<查询JSON(pageNum=N)>
pageNum=N & pageSize=20
turnpage=<hidTurnPage令牌>
sortField=PT & sortType=desc & dstyle=listmode
productStr & searchFrom & ...
```

**关键发现**:
1. `turnpage` 令牌从 HTML 响应的 `<input id="hidTurnPage">` 提取
2. 翻页需要 18 个完整 POST 参数，缺一不可
3. `boolSearch=true` 时知网强制 CurPage="1"，翻页必须用 `boolSearch=false`
4. **v2.3**: 翻页时每页的详情链接按论文顺序累积，传入采集阶段，解决第 2+ 页论文无法获取详情的问题

### 3.3 LLM 提取器（extractor.py — V1 复用）

将非结构化文本通过 DeepSeek 提取为 16 个结构化字段（姓名、别名、性别、民族、学历、国籍、生卒年、籍贯、活动领域、受教育机构、在职单位、职业、职称、电子邮箱、著作实体、控制号）。

多来源优先级（防数据污染）:
```
LLM自适应提取 (20) > 官网个人页 (10) > 期刊官网 (9) > 百度百科 (8) >
论文HTML全文 (7) > 论文摘要页 (5) > 搜索引擎摘要 (3) > 搜索条件 (1)
```

### 3.4 提取策略（record.py — V1 复用）

- **防合著者污染**: 仅从目标姓名所在句子提取
- **增量合并**: 多来源数据按优先级合并

### 3.5 Web 管理界面（V1）

Flask Web 服务（:8820）：搜索启动、流水线监控、记录编辑、Excel 导出。

## 四、技术难点与解决方案

| 难点 | 尝试方案 | 最终方案 |
|------|----------|----------|
| **知网翻页** | pageNum → briefRequest → setQueryJson → ... | **完整 QueryJson + turnpage 令牌 + 18 参数** |
| **第2+页详情缺失** | 从 window.__h 提取（只有第1页） | **翻页时跨页累积 detail_urls** |
| **页面激活** | 手动搜索激活 → 模拟点击 | **手动设置全部搜索参数（无需 UI 激活）** |
| **反爬检测** | DrissionPage → Playwright | CDP 代理, `navigator.webdriver=false` |
| **DOM 注入** | innerHTML 丢失隐藏 input | **Python 正则从原始 HTML 提取** |
